# IRM Framework 실행 가이드

> IRM 문서를 Claude 프로젝트에 올리고, 아래 프롬프트를 순서대로 실행하면 됩니다.
> 최종 갱신: 2026-03 v10

---

## 준비

1. Claude 프로젝트 생성
2. IRM 문서를 프로젝트 지식에 업로드
3. 도메인과 상황을 정하고 Phase 0부터 시작

---

## Phase 0 — 현황 파악 (소요: 5분)

**뭘 하나:** 우리 조직의 리소스(문서/시스템/절차)를 전수 조사하고, 예상 질의를 만들어 행위 유형별로 분류합니다.

```
IRM Phase 0 수행해줘. Step 1→2→3 전체.

도메인: [조직/팀 설명]
상황: [규모, 현재 상황, 통합하고 싶은 업무]
인프라: [사용 중인 DB/시스템]
```

**예시:**
```
IRM Phase 0 수행해줘. Step 1→2→3 전체.

도메인: 중견 이커머스 기업 운영 통합 에이전트
상황: 자사몰 운영, 월 매출 50억, 운영팀 20명.
      상품관리/주문처리/재고관리/배송관리/프로모션/고객분석을 통합하고 싶다.
인프라: PostgreSQL + pgvector.
```

**결과물:** 리소스 인벤토리, 리소스 성격 분류표, IRM 매핑표 (질의 → 행위 유형)

---

## Phase 1 — 설계 (소요: 10분)

**뭘 하나:** 탐색/접근/실행(확정)/실행(유연) 각 영역의 상세 요구사항을 도출합니다.

```
Phase 0 결과 기반으로 Phase 1 전체 수행해줘.
1-A ~ 1-E 전체 (-1, -2 포함).
인프라: [사용 중인 DB/시스템]
```

**결과물:** RAG 컬렉션 후보, CLI 대상 시스템, 실행 절차 목록, 체이닝 관계 맵

---

## Phase 2 — 구현 설계 (소요: 영역별 10~15분)

**뭘 하나:** 각 영역의 기술 구현을 설계합니다. 한꺼번에 하지 말고 영역별로 나눠서 실행합니다.

**RAG 설계:**
```
Phase 1 결과 기반으로 Phase 2 RAG 수행해줘.
탐색 구현 체크리스트 §1~§8.
인프라: [벡터DB 종류]
```

**접근(CLI+REST) 설계:**
```
Phase 2 접근 구현 수행해줘.
접근 구현 체크리스트 1~5.
```

**실행 설계:**
```
Phase 2 실행 구현 수행해줘.
실행(확정) + 실행(유연) 구현 체크리스트.
입력: 1-C, 1-D, 1-E 결과.
```

**결과물:** RAG 컬렉션/임베딩/검색 전략, CLI+REST 접근 설계, 실행 구조/판단 프레임워크

---

## 전체 소요 시간

| 단계 | 소요 | 누적 |
|------|:---:|:---:|
| Phase 0 (현황 파악) | 5분 | 5분 |
| Phase 1 (설계) | 10분 | 15분 |
| Phase 2 RAG | 10분 | 25분 |
| Phase 2 접근 (CLI+REST) | 10분 | 35분 |
| Phase 2 실행 | 15분 | 50분 |

**약 50분이면 AI 에이전트의 전체 기술 설계가 완료됩니다.**

## 검증 완료 도메인

| 도메인 | 인프라 | 핵심 특성 |
|--------|-------|---------|
| LLM 거버넌스 | Qdrant | 탐색 중심, 정책 관리 |
| DevOps 기술운영 | Qdrant (EKS) | 접근 heavy, 인시던트 대응 |
| CS 고객지원 | Qdrant | 탐색+외부 heavy, 채널 분기 |
| HR 인사관리 | pgvector | 실행 heavy, 법적 리스크 |
| 이커머스 운영 | pgvector | 접근+실행 균형, CRUD 중심 |
| 재무/구매 승인 | pgvector | 실행(확정) 최다, GraphRAG 승격 대상 |
| HR 온보딩 | pgvector | 실행(확정)+탐색 룩업, GraphRAG 불필요 |
| SaaS 고객 성공팀 | pgvector | 탐색+실행(유연), GraphRAG 경계선 |
