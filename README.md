# IRM (Intent Routing Map) Framework

AI 에이전트 시스템 설계 시, 고객 요청의 의도를 **행위 유형**으로 분류하고, 성숙도에 따라 적절한 기술로 매핑하는 프레임워크.

---

## 핵심 아이디어

기존 AI 에이전트 설계는 "데이터가 어디 있느냐"에 따라 기술을 선택했다. IRM은 **고객의 의도**에 따라 기술 경로를 결정한다.

```
"지금 매출이 얼마야?"     → 접근 (위치 아는 데이터 조회)
"매출이 왜 떨어졌어?"     → 탐색 (과거 문서/이력 검색)
"매출 보고서 만들어줘"    → 접근 + 실행 (조회 후 생성)
```

모든 분류는 세 가지 질문으로 결정된다:
1. 데이터 위치를 아는가? → 탐색 / 접근
2. 처리 절차가 있는가? → 실행
3. 절차가 결정적이고 재사용 가능한가? → 실행(확정) / 실행(유연)

---

## 구조

```
IRM Framework v10
├── Phase 0: 분류 — 리소스 인벤토리 → 성격 분류 → 질의 매핑
├── Phase 1: 배치 — 탐색/접근/실행별 요구사항 도출
├── Phase 2: 구현 — 체크리스트 + 별첨 방법론
└── 운영 — 매핑표 유지보수, 승격/강등
```

| 행위 유형 | 초기 구현 | 표준화 시 |
|----------|---------|---------|
| 탐색(내부) | RAG (pgvector) | → GraphRAG (pgvector + Apache AGE) |
| 탐색(외부) | 웹 검색 / 크롤링 | 웹 검색 / 크롤링 |
| 접근 | 직접 API/도구 호출 | → CLI + REST API |
| 실행(확정) | 프롬프트 | → CLI 파이프라인형 |
| 실행(유연) | 프롬프트 | → CLI 에이전트형 |
