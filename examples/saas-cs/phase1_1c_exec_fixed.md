## 1-C-1: 절차 목록

| # | 절차명 | 트리거 조건 | 절차 요약 | 관련 접근 시스템 | 현재 단계 |

| 19 | 계정 정지·해제 | "계정 정지/해제해줘" + 고객ID + 사유 | 계정 상태 확인 → 정지/해제 실행 → 사유 기록 → 알림 발송 | Admin API(#16), Gmail(#13) | 프롬프트 |

| 22 | 인보이스 오류 처리 | "인보이스 오류 처리해줘" + 인보이스ID | 인보이스 조회 → 오류 확인 → 수정 → 재발행 → 안내 메일 | Chargebee(#10), Gmail(#13) | 프롬프트 |

> **v7 분기 기준 적용:**
> - 위 5개 절차는 모두 외부 시스템 실행(API 호출)을 포함한다.
> - → 성숙 시 **MCP 파이프라인형** 경로. Skills(프롬프트만)로 승격하지 않는다.
> - 순서가 고정되어 있고 LLM 판단이 최소화되는 구간이 핵심 특징.

## 1-C-2: 실행(확정) 요구사항 도출

질의 매핑표에서 각 절차의 입력/출력 패턴과 실행 조건을 역추적한다.

| 절차 | 필수 입력 | 필수 출력 | 실패 조건 (처리 중단) | MCP 파이프라인 승격 우선순위 |
|-----|---------|---------|-----------------|:----------------------:|
| 신규 고객 온보딩 (#17) | 고객ID, 플랜명 | 온보딩 체크리스트 생성 결과, 환영 메일 발송 여부 | CRM에 고객 미존재, 이미 온보딩 완료 상태 | ★★★ |
| 구독 갱신·해지 (#18) | 고객ID, 처리 유형(갱신/해지) | 처리 결과, 구독 상태 변경 확인, 메일 발송 여부 | 이미 해지 상태(중복), 미결 인보이스 존재 시 해지 불가 | ★★★ |
| 인보이스 오류 처리 (#22) | 인보이스ID (또는 고객ID) | 수정된 인보이스 ID, 재발행 확인, 안내 메일 발송 여부 | 인보이스 미존재, 이미 결제 완료 상태 | ★★ |
| 계정 정지·해제 (#19) | 고객ID, 처리 유형(정지/해제), 사유 | 계정 상태 변경 확인, 알림 발송 여부 | 이미 정지 상태(중복 정지), 사유 누락 | ★★ |
| 장애 에스컬레이션 (#21) | 티켓ID, 에스컬레이션 레벨 | Slack 알림 발송 확인, 티켓 상태 변경 확인 | 티켓 미존재, 이미 최고 레벨 에스컬레이션 상태 | ★★ |

## v7 분기: MCP 파이프라인형 vs Skills

실행(확정) 절차를 성숙 시 어느 경로로 올릴지 판단한다.

| 절차 | 외부 시스템 실행 포함? | → 성숙 경로 | 판단 근거 |
|-----|:----------------:|-----------|---------|
| 신규 고객 온보딩 (#17) | O (CRM 업데이트, Gmail 발송) | MCP 파이프라인형 | API 호출 포함, 순서 고정 |
| 구독 갱신·해지 (#18) | O (Chargebee 처리, Gmail 발송) | MCP 파이프라인형 | API 호출 포함, 순서 고정 |
| 인보이스 오류 처리 (#22) | O (Chargebee 수정/재발행, Gmail 발송) | MCP 파이프라인형 | API 호출 포함, 순서 고정 |
| 계정 정지·해제 (#19) | O (Admin API 실행, Gmail 발송) | MCP 파이프라인형 | API 호출 포함, 단 계정 정지는 권한 분리 필요 |
| 장애 에스컬레이션 (#21) | O (Zendesk 상태 변경, Slack 알림) | MCP 파이프라인형 | API 호출 포함, 에스컬레이션 순서 고정 |

> **5개 전부 MCP 파이프라인형 경로.** 이 도메인에서 "순수 프롬프트(프로세스 안내만)"로 끝나는 실행(확정)은 없다. 모두 외부 시스템 API 호출을 포함한다.

## 성숙도 메모

| 항목 | 초기 구현 | 성숙 시 |
|-----|---------|--------|
| 온보딩, 갱신·해지 | 프롬프트 + 직접 API 호출 | MCP 파이프라인형 — 1-B의 CRM·Chargebee MCP와 연결 |
| 인보이스 오류 처리 | 프롬프트 + 직접 API 호출 | MCP 파이프라인형 — Chargebee MCP와 연결 |
| 계정 정지·해제 | 프롬프트 + 직접 API 호출 | MCP 파이프라인형 — Admin API MCP와 연결, **단 계정 정지는 승인 단계 필요 여부 별도 검토** |
| 장애 에스컬레이션 | 프롬프트 + 직접 API 호출 | MCP 파이프라인형 — Zendesk·Slack MCP와 연결 |

> **계정 정지 주의:** 계정 정지는 고객 서비스 차단 효과가 있어 오발동 시 임팩트가 크다. 자동 실행 전에 "정지 사유 + 확인" 단계를 프롬프트로 유지할지, MCP 내에 승인 게이트를 둘지 현업 검토 필요.
