# Phase 1 1-C — 실행(확정) 배치

> 도메인: 제조기업 — 검사 절차 자동화
> 단계: Phase 1 1-C
> 상태: 완료
> 입력: phase0_step3_query_mapping.md (실행(확정) 질의 #7~10)
> 기준: IRM Framework v10

---

> **v10 실행(확정) 승격 경로:**
> ```
> 초기: 프롬프트 (LLM이 절차를 안내)
>     ↓ 외부 시스템 실행+추적 필요해지면
> 고도화: CLI 파이프라인형
> ```
> 이 유즈케이스는 외부 시스템 실행이 포함되므로 **즉시 CLI 파이프라인형**.

## 1-C-2: 실행(확정) 요구사항 도출

| # | 절차명 | 입력 | 출력 | 단계 수 | 관련 시스템 | 승격 우선순위 |
|---|-------|------|------|:------:|-----------|:----------:|
| 1 | 수입검사 | 자재 로트 ID | 합격/불합격 + 기록 | 5단계 | ERP, LIMS, 품질 DB | ★★★ |
| 2 | 공정검사 | 공정 ID, 로트 ID | 합격/불합격 + 기록 | 4단계 | MES, 품질 DB | ★★★ |
| 3 | 출하검사 | 제품 로트 ID | 출하 판정 + 기록 | 5단계 | 품질 DB, LIMS | ★★★ |
| 4 | 검사 성적서 | 검사 결과 ID | PDF 성적서 | 3단계 | 품질 DB, LIMS | ★★ |

### 수입검사 파이프라인

```
1. erp material-receipt --lot {lot}     → 입고 정보 확인
2. 탐색: 검사 기준서에서 해당 자재 검사 항목 (기존 RAG)
3. lims request-test --sample {lot}     → 시험 요청 (LIMS 있는 경우)
4. 합격/불합격 판정 (기준 대조 — 자동)
5. quality record-result --lot {lot} --result {pass/fail}  → 기록
```

### 공정검사 파이프라인

```
1. mes process-data --process {id}      → 공정 데이터 조회
2. 탐색: 검사 기준서에서 해당 공정 검사 항목 (기존 RAG)
3. 합격/불합격 판정 (기준 대조 — 자동)
4. quality record-result --lot {lot} --result {pass/fail}  → 기록
```

### 출하검사 파이프라인

```
1. quality inspect-result --lot {lot}   → 전 공정 검사 이력 확인
2. lims test-result --sample {lot}      → 최종 시험 결과
3. 탐색: 출하 기준서 (기존 RAG)
4. 출하 판정 (합격/조건부/불합격 — 기준 대조)
5. quality record-result --lot {lot} --result {pass/conditional/fail}
```

---

> **유즈케이스 1과 완전히 다른 구조:**
> - 유즈케이스 1: 탐색 → LLM 판단 (실행 유연)
> - 유즈케이스 2: 접근 → 탐색(룩업) → 기준 대조 → 기록 (실행 확정)
> LLM 판단이 아니라 **규칙 기반 파이프라인**.
