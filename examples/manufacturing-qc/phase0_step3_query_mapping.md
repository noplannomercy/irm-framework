## 주목할 패턴

**1. 탐색(내부) 질의에서 인과 관계 패턴이 뚜렷하다**
#6 "비슷한 불량 전에 있었어?", #7 "이 불량 원인이 뭐였어?" — 불량→원인→공정→설비 인과 체인 탐색.
→ GraphRAG 승격 검토 시그널.

**2. "불량"이 다의어다**
수입불량(자재 결함), 공정불량(제조 결함), 출하불량(유출 결함), 고객불량(클레임) — 같은 "불량"이지만 맥락이 완전히 다르다.
→ GraphRAG 승격 검토 시그널.

**3. 실행(유연) 질의가 접근+탐색과 항상 결합된다**
#21~24 모두 "데이터 조회(접근) + 이력/기준 탐색 + 판단" 패턴. 체이닝이 필수적.

**4. 검사 3종(수입/공정/출하)이 실행(확정)의 핵심**
#17~19 모두 순서 고정, 합격/불합격 판정 명확. CLI 파이프라인형 후보.
