## 1-C-2: 실행(확정) 요구사항 도출

| # | 절차명 | 입력 | 출력 | 단계 수 | 관련 시스템 | 승격 우선순위 |
|---|-------|------|------|:------:|-----------|:----------:|
| 1 | 수입검사 | 자재 로트 ID | 합격/불합격 판정 + 검사 기록 | 5단계 | ERP, LIMS, 품질DB | ★★★ |
| 2 | 공정검사 | 공정 ID, 로트 ID | 합격/불합격 + SPC 데이터 기록 | 4단계 | MES, SPC, 품질DB | ★★★ |
| 3 | 출하검사 | 제품 로트 ID | 출하 판정 (합격/조건부/불합격) | 5단계 | 품질DB, LIMS, DMS | ★★★ |
| 4 | 검사 성적서 발행 | 검사 결과 ID | PDF 성적서 | 3단계 | 품질DB, LIMS, DMS | ★★ |

### 수입검사 파이프라인 예시

```
1. ERP에서 입고 자재 정보 조회 (erp material-receipt --lot {lot})
2. 검사 기준서에서 해당 자재 검사 항목 확인 (탐색)
3. LIMS에 시험 요청 또는 자체 검사 수행 (lims request-test --sample {lot})
4. 검사 결과 판정 (합격/불합격 기준 대조)
5. 품질DB에 결과 기록 (quality record-result --lot {lot} --result {pass/fail})
```
