> 도메인: 제조기업 — 품질 불량 원인 분석
> 단계: Phase 1 1-D
> 상태: 완료
> 입력: phase0_step3_query_mapping.md (실행(유연) 질의 #10, #13)
> 기준: IRM Framework v10

---

> **v10 실행(유연) 승격 경로:**
> ```
> 초기: 프롬프트 (LLM이 상황 판단하며 실행)
>     ↓ 외부 시스템 실행+추적 필요해지면
> 고도화: CLI 에이전트형 (REST 게이트웨이)
> ```
> 단, 이 고객은 온프레미스이므로 REST 게이트웨이는 사내망 내부.

| # | 절차명 | LLM 판단 필요 지점 | 가변 요소 | 전환 조건 |

### 불량 원인 분석 흐름

```
입력: "이 불량 원인 분석해줘" + 불량 정보
  ↓
1. 접근 — 품질 DB에서 해당 불량 상세 조회
  ↓
2. 접근 — MES에서 해당 공정 조건 조회
  ↓
3. 탐색 — 유사 불량 이력 검색 (GraphRAG: 불량→원인→공정→설비 체인)
  ↓
4. 탐색 — 검사 기준서에서 해당 기준 확인
  ↓
5. 탐색 — 공정 매뉴얼에서 관련 작업 조건 확인
  ↓
6. LLM 판단 — 수집된 정보로 원인 후보 제시
  ↓
7. 사람 판단 — 최종 원인 확정 (human-in-the-loop)
```

> **human-in-the-loop:** LLM은 원인 후보를 제시하고, 최종 확정은 품질팀이 한다.
