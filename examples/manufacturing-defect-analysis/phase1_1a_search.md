## GraphRAG 승격 판단 (v10)

> v10 기준: 4개 조건 중 **2개 이상 충족 시** GraphRAG 승격을 검토한다.

| # | 승격 조건 | 충족 여부 | 근거 |
|---|---------|:-------:|------|
| 1 | 동음이의어/중의어가 많다 | **O** | "버 발생" vs "성형 불량" — 같은 불량인데 표현이 다름. ALFA 미팅에서 직접 확인 |
| 2 | 관계/인과 기반 질의 패턴 발견 | **O** | #5 "원인이 뭐였어?", #10 "원인 분석해줘", #11 "같은 원인 불량", #12 "설비 관계" — 탐색 질의 중 인과 패턴 50% |
| 3 | RAG 히트율 목표치 미달 | — | 운영 전 판단 불가. 단, 이전 PoC에서 정확도 이슈 경험 있음 |
| 4 | 동일 엔티티가 여러 맥락으로 등장 | **O** | "불량"이 수입불량/공정불량/출하불량으로, "기준"이 검사기준/관리기준/판정기준으로 분산 |

**충족: 3/4** → **GraphRAG 승격 대상**

**판단:** 이 유즈케이스는 인과관계 추적이 핵심이다. 불량→원인→공정→설비 체인을 그래프로 탐색하면 "비슷한 불량"을 관계 구조로 찾을 수 있다. 파일럿에서 기본 RAG 먼저 돌리고, 표현 비통일 문제가 심하면 GraphRAG 승격.

> **참조:** IRM Framework v10 — 성숙도 승격 기준 / GraphRAG 설계 방법론 v10

## 컬렉션 설계 제안

```
quality-docs       — 검사 기준서, 공정 매뉴얼
                     갱신: 기준/공정 변경 시 / 최신형
                     메타데이터: product, process, inspection_type, equipment

defect-history     — 불량 이력 텍스트 (품질 DB에서 추출)
                     갱신: 불량 발생 시 / 이력형 (3년+)
                     메타데이터: defect_type, product, root_cause, process, period

equipment-docs     — 설비 관련 자료
                     갱신: 설비 변경 시 / 이력형
                     메타데이터: equipment, equipment_type
```
