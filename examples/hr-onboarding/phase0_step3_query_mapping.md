## 주목할 패턴

**1. 실행(확정) 질의가 8개 — SaaS CS(5개) 대비 훨씬 많다**
#13~19, #25 모두 순서 고정, 입력 명확, 재사용 가능. MCP 파이프라인형 후보 다수.

**2. Day1 온보딩 전체 (#19)가 모든 실행(확정)을 통합 진입점으로 묶는다**
계정 생성(#13) + 장비 배포(#14) + 권한 배정(#15) + 메일 발송(#16) + 교육 등록(#17) + 서류 수집(#18) 전부 포함.
→ Day1 파이프라인이 핵심 MCP 설계 대상.

**3. 접근 단독 질의(#1~5)는 모두 입사자 ID 기반**
BambooHR + Okta + AssetDB + AD/LDAP — 항상 입사자 ID를 기준으로 조회.
→ SaaS CS의 "고객 컨텍스트 패키지"에 대응하는 **"입사자 컨텍스트 패키지"** 패턴이 보인다.

**4. 퇴사 처리 (#25)가 실행(확정)으로 식별됨**
온보딩 역방향 절차(계정 삭제, 장비 반납, 권한 해제). 온보딩 MCP와 대칭 구조.
→ 같은 서버에서 onboarding/offboarding 양방향 처리 가능.
