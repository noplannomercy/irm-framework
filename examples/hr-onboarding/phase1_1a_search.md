## GraphRAG 승격 판단 (v8)

> v8 기준: 4개 조건 중 **2개 이상 충족 시** GraphRAG 승격을 검토한다.

| # | 승격 조건 | 충족 여부 | 근거 |
|---|---------|:-------:|------|
| 1 | 동음이의어/중의어가 많다 | **X** | 온보딩 도메인의 용어("계정", "장비", "교육", "권한")가 단일 의미로 사용됨. 도메인 간 용어 충돌 없음 |
| 2 | 관계/인과 기반 질의 패턴 발견 | **X** | 탐색 질의 #6~12가 모두 룩업형 ("~알려줘", "~어떻게 돼?"). 인과 체인 탐색이 필요한 질의 없음 |
| 3 | RAG 히트율 목표치 미달 | — | 운영 전이므로 판단 불가 |
| 4 | 동일 엔티티가 여러 도메인에 다른 맥락으로 등장 | **X** | 엔티티가 HR 도메인 내에서 명확히 분리됨. "온보딩 정책", "장비 기준", "보안 정책" 등 혼동 여지 없음 |

**충족: 0/4** → **승격 불필요**

**판단:** RAG(pgvector 단독)로 충분하다. 운영 중 부서 간 정책 충돌 질의가 증가하거나, 조직 규모 확대로 도메인 간 용어 중복이 발생하는 경우 재검토.

> **참조:** IRM Framework v8 — 성숙도 승격 기준 / GraphRAG 설계 방법론 v1

## 컬렉션 설계 제안

```
hr-policy      — 온보딩 정책, 보안 정책, 복리후생, 근로계약 FAQ
                 갱신: 정책 변경 시 / 검색 패턴: 정책 유형·적용 범위 기준
                 메타데이터: doc_type, scope, version

dept-guide     — 부서별 온보딩 가이드, 조직도, 부서 안내
                 갱신: 조직/부서 변경 시 / 검색 패턴: 부서명·직군 기준
                 메타데이터: dept, job_type, onboarding_stage

training       — 교육 자료, 필수 이수 목록
                 갱신: 교육 과정 변경 시 / 검색 패턴: 교육 유형·필수 여부 기준
                 메타데이터: training_type, required, job_type

runbook        — 절차 문서 #17~22 (지식 용도)
                 갱신: 절차 변경 시 / 검색 패턴: 절차명·트리거 기준
                 메타데이터: procedure_name, trigger_condition, job_type
```

> **4개 컬렉션 기준:** 갱신 주기(정책/조직/교육/절차)와 검색 패턴(정책조회/부서맞춤/교육/절차참조)이 컬렉션마다 다르다.
