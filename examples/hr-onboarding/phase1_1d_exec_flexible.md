> 도메인: 신규 직원 온보딩 (HR)
> 단계: Phase 1 1-D
> 상태: 완료
> 입력: phase0_step3_query_mapping.md (실행(유연) 포함 질의 #20~24)

---

## 개요

실행(유연)으로 분류된 절차 5개. SaaS CS(4개)와 비슷한 규모이나, 성격이 다르다.
이 도메인의 유연 절차는 "고객 상황 판단"이 아닌 **"직군·상황별 맞춤 처리"** 중심.

## Skills 승격 후보

**맞춤 온보딩 플랜 수립 (#21)**
- 외부 시스템 실행 없음 (조회 후 플랜 문서 생성)
- 출력 포맷이 안정화되면 Skills 승격 가능
- Skills 형태: "직군/부서/경력 기반으로 온보딩 플랜을 구성하라"는 프롬프트 + 출력 형식 표준화
→ **유일한 Skills 승격 후보**

나머지 4개(#20, #22, #23, #24)는 모두 외부 시스템 실행 또는 상태 변경을 포함하므로 패턴 성숙 시 MCP 에이전트형으로 전환.
