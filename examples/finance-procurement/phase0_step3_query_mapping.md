## 주목할 패턴

**1. 실행(확정) 질의가 7개 — 전 도메인 최다 수준**
#15~21 모두 절차 순서 고정, 입력 명확, 재사용 가능. MCP 파이프라인형 후보 다수.

**2. 구매 전체 프로세스 (#21)가 메타 파이프라인**
구매 요청(#15) + 발주서 발행(#16) + 검수(#17) + 지급(#18) 전부 포함.
→ HR 온보딩의 Day1과 동일한 계층적 파이프라인 구조.

**3. 전자결재(#10)가 실행(확정) 4개에 필수 참여**
#15(구매 요청), #18(지급), #20(결재 상신), #21(전체) 모두 전자결재 포함.
→ 전자결재는 독립 MCP 도구가 아니라 파이프라인 내 **승인 게이트 단계**로 설계해야 함.

**4. ERP(#9) + 전자결재(#10) + 회계(#11) 트리오 — Aggregator 후보**
#15, #18, #21, #24 에서 세 시스템이 함께 호출됨.
→ "구매 처리 컨텍스트 패키지" Aggregator 패턴 적용 검토 대상.

**5. 접근 단독(#1~6)은 모두 구매 건 ID 또는 카드 ID 기반 조회**
→ SaaS CS의 "고객 컨텍스트 패키지", HR의 "입사자 컨텍스트 패키지"에 대응하는
   **"구매 컨텍스트 패키지"** 패턴이 보인다.
